# 📊 제네시스 투자 성과 리뷰 워크플로우

사용자가 **"제네시스 하자"** 라고 하면 아래 프로세스를 시작합니다.

## 1. 기준 리포트 탐색 (Source Report Discovery)

브라우저로 `https://stockanalysis2.pages.dev/picks` 에 접속하여 리포트 목록을 확인합니다.

### 🎯 핵심 원칙
웹사이트의 유망 종목은 **단기 / 중기 / 장기** 3가지로 구분되어 있으며, 각 예측 기간이 다르므로 **기간에 맞는 시점에 복기**해야 합니다.

| 구분 | 예측 기간 | 복기 기준 시점 |
|---|---|---|
| 🚀 단기 | ~7일 이내 | 작업 당일 기준 **T-7** 리포트 |
| 📈 중기 | 3~6개월 | 작업 당일 기준 **T-30** 리포트 |
| 🧘 장기 | 1~3년 | 작업 당일 기준 **T-90** 리포트 |

### 🔍 탐색 순서

1. `https://stockanalysis2.pages.dev/picks` 에서 리포트 목록 전체를 확인합니다.
2. 각 리포트의 **종목 구분(단기/중기/장기)** 을 식별합니다.
3. 아래 순서로 복기 대상을 선별합니다.
   - **단기 복기**: 작업 당일 기준 약 7일 전에 작성된 **단기 유망 종목** 리포트를 탐색합니다. 단기 종목은 7일간의 예측이므로, 반드시 7일 안팎의 리포트여야만 의미 있는 복기가 됩니다.
   - **중기 복기**: 작업 당일 기준 약 30일 전에 작성된 **중기 유망 종목** 리포트를 탐색합니다.
   - **장기 복기**: 작업 당일 기준 약 90일 전에 작성된 **장기 유망 종목** 리포트를 탐색합니다.
4. 해당 기간의 리포트가 없으면 **해당 구분의 복기를 생략**합니다. (예: 90일 전 장기 리포트가 없으면 장기 복기 없이 단기·중기만 진행)
5. 복기 대상이 하나 이상 확인되면 다음 단계로 진행합니다.

> ⚠️ **주의**: 단기 유망 종목을 30일이나 90일 후에 복기하는 것은 의미가 없습니다. 예측 기간을 벗어난 종목은 복기 대상에서 제외합니다.

## 2. 현재 주가 수집 (Real-time Price Fetching)

> 💡 **`market-data-miner-skill` 활성화** — 이 단계에서 데이터 수집 및 Failover 처리를 담당합니다.

기준 리포트에서 추천 종목 목록(ticker, 추천가)을 추출한 뒤 아래 우선순위로 현재가를 수집합니다.

- **Priority 1: 네이버 증권** (`finance.naver.com/item/main.naver?code=XXXXXX`): 현재가, 등락률, 수급 데이터
- **Priority 2: 토스증권**: 네이버 접근 불가 시 대체
- **미국 주식**: 구글 파이낸스에서 현재가 및 USD/KRW 환율 반영
- **Failover**: 스크래핑 차단 시 차트 캡처 후 Vision 분석으로 수치 역산

## 3. 성과 계산 및 분석 (Performance Analysis)

> 💡 **`genesis-analyst-skill` 활성화** — 이 단계부터 스킬의 4가지 분석 모듈이 순서대로 실행됩니다.

각 종목에 대해 아래 지표를 산출합니다.

| 항목 | 계산 방법 |
|---|---|
| 수익률 | `(현재가 - 추천가) / 추천가 × 100` |
| 상태 | WIN(+5% 초과) / HOLD / LOSS(-5% 이하) / EXIT(손절가 이탈) |
| 손절 여부 | 추천 보고서의 손절가 기준으로 판정 |

스킬의 4가지 분석 모듈을 순서대로 실행합니다.

- **[Module 1] 기술적 성과 복기 (Technical Audit)**: 추천 당시 차트 위치 vs 현재 위치, 매수 타점의 적절성, 지지·저항선 작동 여부 판독
- **[Module 2] 논리 유효성 검증 (Logical Validation)**: 추천 Catalyst가 실제 시세 분출의 원인이었는지 대조, 외부 변수 개입 여부 확인, 진입 타이밍의 '시차' 분석
- **[Module 3] 패턴 추출 (Pattern Recognition)**: 반복되는 실수 또는 고승률 패턴 추출, 황원장님 투자 스타일의 강·약점 데이터화
- **[Module 4] 액션 플랜 제안 (Future Action Plan)**: WIN → Trailing Stop 또는 비중 확대 / LOSS → 손절의 적절성 평가 및 진입 금지 패턴 등록 / HOLD → 기회비용 대비 교체 매매 여부 판단

## 4. 성과 보고서 생성 (MDX Report Generation)

분석 결과를 아래 형식의 MDX 파일로 작성합니다.

- **저장 경로**: `content/picks-feedback/`
- **파일명 규칙**: `YYYY-MM-DD-performance-review.mdx`
- **Frontmatter 필수 항목**:
  - `title`: "YYYY-MM-DD 제네시스 성과 분석 리포트"
  - `date`: 작성 날짜 (YYYY-MM-DD)
  - `summary`: 핵심 성과 요약 (1~2문장)
  - `tags`: ["성과분석", "제네시스", "복리전략"]
  - `thumbnail`: Unsplash 차트 이미지 URL

- **보고서 필수 섹션 (SOP v2.0 기준)**:
  1. **[Executive Summary]**: 종목별 수익률 + 평균 수익률 + 승률 + MDD (Module 1 결과)
  2. **[Logic Breakdown]**: 추천 당시 논리 vs 실제 시장의 움직임 대조 (Module 2 결과)
  3. **[Critical Feedback]**: 가장 잘된 점 1가지 + 가장 아쉬운 점 1가지 (Module 3 패턴 기반)
  4. **[Compounding Insight]**: 이번 복기를 통해 배운 '다음 번에 반드시 적용할 한 가지 룰' (Module 4 기반)
  5. **[안팀장의 피드백]**: 황원장님께 드리는 종합 코멘트 및 다음 종목 선정 방향

## 5. 배포 승인 및 Push (Review & Approval)

- 보고서 작성 완료 후 **즉시 황원장님께 내용 보고**

# 🚀 제네시스(Genesis) v3.4 시황 분석 워크플로우

본 워크플로우는 실시간 금융 데이터(네이버, 토스, 구글)를 수집하여 **'3계층 입체 분석'**을 수행하고, 웹사이트 게시용 MDX 리포트를 생성하는 전문 프로세스입니다.

## 1. 실행 트리거 (Trigger)
- 사용자가 **"제네시스 하자"** 혹은 **"제네시스 시황 분석"** 입력 시 가동

## 2. 분석 엔진 가동 (6-Step Engine)

### [Step 1] 실시간 데이터 스캔 (Data Sourcing)
- **기본 소스:** 네이버 증권, 토스 증권, 구글 파이낸스를 기본으로 실시간 데이터를 스캔하며, 필요시 검증받은 다른 사이트를 추가로 이용할 수 있음
- **주요 지수:** 코스피(KOSPI), 코스닥(KOSDAQ) 현재가 및 등락률
- **수급 현황:** 외국인/기관 순매수액 (코스피/코스닥 각각 확인)
- **섹터 모멘텀:** 토스증권 실시간 테마 데이터 및 **네이버 증권의 업종별 시세** 기반 강세 섹터 추출
- **지표 분석:** 원/달러 환율, 미 10년물 금리, **국제 유가**, 공포지수(Fear & Greed) 필수 수집
- **데이터 검증:** 단순히 테마 검색에 그치지 않고, 반드시 **장 마감 확정 등락률(%)**을 대조하여 실질적인 수급이 확인된 종목만 선별함

### [Step 2] 층별 입체 분석 (3-Layer Analysis)
- **[Layer 1] 거시 분석 (Macro):** 환율 변동성 및 거시 리스크가 증시에 미치는 영향 분석
- **[Layer 2] 중기 분석 (Mid-term):** 지수 바닥 확인, 수급 쏠림 현상, 섹터 로테이션 패턴 진단
- **[Layer 3] 단기 분석 (Short-term):** 당일 주도 섹터 및 **상한가/급등주(대장주)** 중심의 수급 분석. 등락률이 미미한 후발주자나 기대감뿐인 소외 테마는 철저히 배제함.

### [Step 3] 리스크 판정 (Risk Assessment)
- 현재 시장의 결정적 상방/하방 트리거 규정
- 과매도/과매수 구간 판정 및 심리적 임도 분석

### [Step 4] 오늘의 매매 전략 (Action Plan)
- **Do (해야 할 것):** 기관 추종, 분할 매수, 특정 섹터 집중 등 구체적 지침. **반드시 당일 수급과 등락률이 확인된 '구체적인 대표 종목'을 명시할 것.**
- **Don't (하지 말아야 할 것):** 뇌동매매 금지, 낙폭 과대주 추격 금지 등 경고 사항. **리스크가 감지되거나 하락 추세인 '구체적인 대표 종목'을 사례로 들어 경고할 것.**

### [Step 5] 안팀장의 한 줄 평 (Insight Synthesis)
- 전체 시황을 관통하는 촌철살인의 분석 멘트 생성 (Premium Persona)

### [Step 6] 보고서 최종 생성 및 배포 승인 (Push Control)
- **로컬 저장:** `content/market-analysis/YYYY-MM-DD-market-analysis-genesis.mdx` 경로에 먼저 생성
- **GitHub Push 승인:** **GitHub Push 여부는 황원장님이 전적으로 결정**하며, 원장님의 명확한 승인(예: "Go", "Push해줘")이 있을 때만 실행함
- **웹사이트 규격 준수:** `title`, `date`, `category: "Genesis"`, `summary`, `tags`, `thumbnail`(Unsplash) 필드 필수 포함 (누락 시 게시판 노출 제외됨)

## 3. 리포트 스타일 가이드
- 어조: 신뢰감 있고 날카로운 전문 투자 전략가 톤
- **데이터 원칙:** 모든 종목과 테마 언급 시 **반드시 등락률(%)을 병기**하여 객관적 근거를 제시할 것
- 시각화: 표, 인용구, 알림창(`[!IMPORTANT]`) 적극 활용
